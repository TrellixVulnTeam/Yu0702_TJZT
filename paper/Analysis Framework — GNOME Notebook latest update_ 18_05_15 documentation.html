Analysis Framework¶

Important

Anyone interested in working on the GNOME data analysis software should read the documentation by Sam Afach, available here, containing meaningful advices and suggestions regarding basic but essential coding style.

Data Analysis Techniques¶

The main objective of the GNOME project is to look for consistent signatures of domain wall crossing among various stations. In order to achieve this goal, we use two analysis techniques originally developped by the LIGO collaboration for gravitational wave search. Those two techniques are the excess power search and coincidence analysis. While both techniques represent the core components of the GNOME analysis, there are many other analysis tools that can be used to either characterize the instruments or to do statistical analysis on the data themselves. You may click on the icon below to access a full list of analysis techniques that you can use on the GNOME data. Please feel free to edit the document and add more ideas that could help improving our analysis.

Multi-user Server¶

(Click on the above logo to access the JupyterHub server)

A GNOME JupyterHub, or multi-user server has been created to allow each member to access the entire available dataset and do direct analysis on it. Members who do not have access to the server but wish to access it should send a request to me. Please note that members who are not part of the GNOME collaboration will not be granted access to the dataset but are free to use our software on their own data. The server can be accessed in two ways, either by accessing the server’s webpage, or from your terminal through SSH:

ssh -X username@budker.uni-mainz.de -p 8022

You can also create an alias on your system PATH to directly connect to the Jupyter server from the terminal using the sshpass command as follows:

sshpass -p "yourpassword" ssh -X yourusername@budker.uni-mainz.de -p 8022

While SSH is very handy for people using UNIX-like operating systems, this can become more complicated for those working on Windows machines. Fortunately, access to a terminal is also possible through the webpage, which means directly from your internet browser! This can be done by clicking on the New tab after login and select Terminal:

You can then use the terminal window to access files and create new Python scripts for your analysis.

Finally, in case you want to copy a repository from your Jupyter Hub account to your computer, for instance if you want to copy an analysis report you made with Jupyter notebook in the GNOME notebook, you can use the scp command on your terminal from the location you want to copy the remote directory to:

scp -r -P 8022 yourusername@budker.uni-mainz.de:/absolute/path/to/your/remote/directory

Large Dataset Analysis¶

The following issue was raised by Wenhao Li by email on December 25, 2017:

After I downloaded the 3-weeks-long data, I find it extremely slow in processing the data. Even simply plotting the sanity check results (which is 1/(512*4) smaller than the magnetic field data) over the run is a painful task. So I would like to know what is the proper workflow in dealing with several giga bytes data? My current workflow is to use python+h5py+pytables to combine all the minute-long hdf5 files into a large giga-bytes .h5 file so that I can do on-disk querying and searching. But it normally takes several hours to combine the data and after the combined data is ready, I couldn’t find an effective way to insert new data points to the table and maintain it. I do everything locally on my laptop (MacBook Pro) because I met with the similar slow processing problem on the jupyter server. Any suggestions and comments on this will be warmly welcomed!
